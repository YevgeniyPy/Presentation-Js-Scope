Understanding “scope” and “closures” is key to learning JavaScript

Global Object

In JavaScript, all global variables and functions are properties of a special object called a global object.

Assigning or reading a global variable, we actually work with the window properties.


							var a = 5; //declaration var creates  window.a property
							alert( window.a ); //5

You can create a variable by explicit assignment in window:


							window.a = 5;
							alert( a ); // 5

Initialization order

  1. The first phase is initialization, preparation for launch.
  2. In the second phase - actually, implementation.

In JavaScript, there is no difference between a declaration outside the block:


							var i;
							{
							  i = 5;
							}

... And inside it:


							i = 5;
							{
							  var i;
							}

There is also no difference between a declaration in a loop and outside it:


							for (var i = 0; i <5; i ++) {};
							
							var i;
							for (i = 0; i <5; i ++) {}

It doesn't matter where and how many times the variable is declared.


							var i = 10;
							for (var i = 0; i <20; i ++) {
							  ...
							}
							var i = 5;

Always current value

The value of a variable from the outer region is always current. It may not be what it was at the time the function was created.

Total: closure

“Understand closures” in JavaScript means to understand the following things:

  • All variables and parameters of functions are properties of the variable object LexicalEnvironment. Each function launch creates a new such object. At the top level, it is the “global object”, in the browser - the window.
  • When created, the function gets the [[Scope]] system property, which refers to the LexicalEnvironment in which it was created.
  • When calling a function, wherever it is passed in the code, it will look for variables first in itself and then in external LexicalEnvironment from the place of its “birth”.

Block Scope

Variable let and const

The scope of the let and const variables is the {...} block.

For example, a variable var:


							var apples = 5;
							if (true) {
							  	var apples = 10
							 	 alert (apples); // 10 (inside the block)
							 }
							alert (apples); // 10 (the same outside the block)

The same with let will work differently:


							let apples = 5;
							if (true) {
							  	let apples = 10
							 	alert (apples); // 10 (inside the block)
							 }
							alert (apples);
							 // 5 (the value has not changed outside the block)

the let and const variables is always visible in the block where it is declared.


							if (true) {
								let apples = 10;
								alert (apples); // 10 (inside the block)
							 }
							alert (apples); // mistake!
